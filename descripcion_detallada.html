Diseño del Sistema Portafolio

Arquitectura

El proyecto sigue una arquitectura simple con separación de responsabilidades:

  • main.go: Punto de entrada, configuración de rutas y carga de proyectos en memoria
  • db/: Manejo de base de datos SQLite para mensajes de contacto
  • cron/: Tareas programadas (ej. envío de emails pendientes)
  • ui/: Plantillas templ para las vistas

Estrategia de Caché

Los proyectos se cargan en memoria al iniciar el servidor usando una estructura:

map[string]map[string][]byte

Donde el primer nivel es el nombre del proyecto y el segundo nivel es la ruta del archivo con su contenido en bytes. Esto permite servir archivos sin leer del disco en cada request.

Seguridad

  • Content Security Policy (CSP) configurado
  • Protección contra spam con campo honeypot
  • Límite de mensajes por IP
  • Validación de formularios
